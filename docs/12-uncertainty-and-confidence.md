# Uncertainty And Confidence

Uncertainty is a first-class output. A disciplined pipeline should be able to say "not enough data" without forcing a stronger conclusion.

Unresolved is a valid result.

## Levels

| Level | Meaning |
| --- | --- |
| Insufficient data | Critical metadata is absent. Reconstruction is not supportable. |
| Low confidence reconstruction | Some estimates are possible but depend heavily on assumptions. |
| Bounded reconstruction | Range, timing, and geometry are bounded enough to produce limited estimates. |
| Multi-sensor supported | More than one sensor supports the event, but reviewability may still be limited. |
| Independently reconstructable | Another reviewer can reproduce estimates from available records. |
| Scientifically reviewable | Data, calibration, provenance, and methods support formal technical review. |

## Confidence Inputs

- Range certainty.
- Sensor calibration.
- Timestamp quality.
- Platform state.
- Track continuity.
- Correlation across sensors.
- Compression and custody.
- Alternative-cause screening.

## Boundary

Confidence applies to the reconstruction, not to preferred narratives. A low-confidence reconstruction cannot support a high-confidence propulsion claim.
