# Synthetic Sensor Generation

Synthetic sensor frames help test pipelines without implying a real event.

Synthetic data is not evidence of a real event. It is a tool for testing pipelines.

## Synthetic Modes

- Simulated optical track.
- Simulated infrared signature.
- Simulated radar return.
- Simulated metadata loss.
- Compression artifacts.
- Clock drift.
- Platform motion.
- Parallax.

## Use Cases

- Validate schema coverage.
- Test missing-metadata classification.
- Demonstrate range-dependent speed estimates.
- Evaluate how parallax can produce apparent acceleration.
- Exercise multi-sensor fusion reports without using sensitive data.

## Boundary

Synthetic examples must be labeled as synthetic, fictional, and public-safe. They must not reference real classified cases, named disputed events, named pilots, whistleblowers, or sensitive sources.
